**Инструкция по созданию релиза и загрузке архивов**

- **Назначение**: Скрипт `scripts/release_and_upload.sh` архивирует только папки `atmosphere`, `config`, `switch`, затем создаёт (или обновляет) релиз в GitHub и загружает архивы как assets.

- **Требования**:
  - `gh` (GitHub CLI) установлен и аутентифицирован (`gh auth login`).
  - `zip` установлен (скрипт использует `zip` для упаковки).
  - Сборка/команда запуска выполняется из корня репозитория (где находятся папки `atmosphere`, `config`, `switch`).

- **Файл скрипта**: `scripts/release_and_upload.sh`

- **Пример использования**:
  - Сделать скрипт исполняемым:

```bash
chmod +x scripts/release_and_upload.sh
```

  - Создать релиз и загрузить архивы (пример):

```bash
scripts/release_and_upload.sh -t v1.0.0 -n "Release v1.0.0" -b "Обновлённые конфиги и atmosphere" -r Dimasick-git/Ryazhenkabestcfw-Tuner
```

  - Если релиз с тегом уже существует, скрипт загрузит (и перезапишет) assets в этот релиз.

- **Параметры**:
  - `-t <tag>` — обязательный, тег релиза (например `v1.0.0`).
  - `-n <release-name>` — название релиза (по умолчанию равно тегу).
  - `-b <body>` — заметки релиза.
  - `-r <repo>` — репозиторий в формате `owner/name` (по умолчанию `Dimasick-git/Ryazhenkabestcfw-Tuner`).
  - `-p <prefix>` — префикс для имени zip-файлов (опционально).

- **Гарантии безопасности**:
  - Скрипт не пушит код; он только создаёт zip-архивы локально и загружает их в релиз через `gh`.

**Автоматизация через GitHub Actions**

- Добавлен workflow: `.github/workflows/version-trigger-build.yml`.
  - Триггер: `push` и `pull_request` при изменениях файлов `**/config.ini`.
  - Детектит: сравнивает значение `version` в изменённых `config.ini` между коммитами.
  - Действие: если `version` изменился, запускается задача `build`, которая архивирует
    папки `atmosphere`, `config`, `switch` и загружает их как артефакт в Actions.

Workflow теперь автоматически создаёт или обновляет релиз в `main` и загружает zip-файлы как assets.

- Поведение релиза:
  - Сначала проверяется, существует ли релиз с тегом `v<version>`; если есть — тело релиза обновляется и артефакты с тем же именем удаляются и перезаписываются.
  - Если релиза нет — он создаётся.
